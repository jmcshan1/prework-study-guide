HTML

  • The <head> element contains information about the webpage.The user cannot see what is contained in the head element.
  • The <body> element represents the visible content shown to the user.
  • The <ul>,<ol>, and <li> represent unordered lists, ordered lists, and list items.
  • The <img> element contains information about images that are displated on the webpage.
  • The <header> element appears at the top of the page and usually contains a navigation menu and/or introductory content.
  • The <footer> element appears at the bottom of the page.
  • The href attribute inside a link element is relative to the current location of the file you are working in.
  • All HTML elements are like boxes. The padding property changes the space around the content inside an element.
  • The id attribute inside a tag is used to identify elements and select them.

CSS

  • The margin indicates how much space we want around the outside of an element.
  • Padding indicates how much space we a want around the content inside an element.
  • Inline CSS styles are applied only to a specific tag and its corrosponding element.
  • Internal CSS style sheets are CSS rules contained in the head of the HTML file that apply to that webpage.
  • External CSS style sheets are a separate CSS file that contains rules. You must link the CSS file to the HTML file for the CSS rules to apply to the webpage
  • A class attribute allows us to add styles to only certain elements that are assigned a class selector.

Git

  • The git status: checks what branch we are currently on
  • The git checkout -b branch-name: creates a new branch and switched to it
  • The git add -A command stages all change in the working branch.The -A part makes it so this applies to all files.
  • Always stage changes before you commit. If you don't stage after every change you make, Git won't record the changes that you make.
  • The git commit command commits all changes that were staged. Adding a -m plus a message in quotes afterwards adds a message along with the changes.
  • Git pull origin main pulls from the remote main branch from github to check if your local branch is up to date. Good to see if there are no comflicts between your local and remote files.
  • Git push origin "branch name" pushes changes from your current local branch to whichever remote Github branch you specified. Always push to your branch and not to the remote branch.

Javascript

  • A variable is a named contained used to store data in our code.
  • An array is a single variable that can hold a group of data. You can store different type of data within an array.
  • An index is the location of a data item in an array.
  • Console.log() adds a message users that is displayed in the web console.
  • Adding <script src="file location> into HTML links that page to the javascript similar to CSS.
  • In javascript, code runs from the first statement at the top of the page to the last statement at the bottom of the page.
  • Conditional statements(if statements) and loops can control the order in which code is executed or if its executed at all.
  • If statements only execute the code inside of them if a condition is met, or is true. If it isn't true, the computer will ignore the code and move on.
